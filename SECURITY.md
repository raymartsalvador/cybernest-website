# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository or in any Cybernest
Solutions service, please **report it privately** rather than opening a
public issue or posting it on social media.

- **Email:** security@cybernestsolution.com
- **Expected first response:** within 3 business days
- **Preferred language:** English

Please include:

1. A description of the vulnerability and the impact you believe it has.
2. Steps to reproduce (a minimal proof-of-concept is ideal).
3. Any affected URLs, endpoints, or versions.
4. Your name/handle if you would like to be credited in the fix notes.

## Scope

In scope:

- This repository (`cybernest-website`) and anything it deploys to
  `https://www.cybernestsolution.com`.
- The same-origin `/api/*` Vercel serverless functions this site exposes.

Out of scope:

- Denial-of-service testing against production.
- Social-engineering or physical attacks against Cybernest staff.
- Findings that require a compromised user device or browser extension.
- Vulnerabilities in third-party services (report those to the vendor).

## Safe Harbor

We will not pursue legal action against researchers who:

- Make a good-faith effort to avoid privacy violations, data destruction,
  and service disruption.
- Do not exploit the issue beyond what is necessary to demonstrate it.
- Give us a reasonable window to remediate before any public disclosure.

Thank you for helping keep Cybernest and our users safe.
